このタグっていったい何?

現在スラッシュドットでは、記事にタグを付ける実験をしています。ぜひこの機能を利用して、一握りのタグを提供してください。タグは記事の内容を最も的確に表していると思われる簡単なラベルです。それが 'セキュリティ' と 'Mozilla' に関する記事であることを伝えることができます。

これはまだまったくの実験段階です。現在のところ、タグ付けはアカウント作成からほぼ半年以上経過しているユーザに開放されています。

これが今後どのような形になるかははっきりしておらず、実際のところ大部分はあなた次第です。独自のタグを追加する場合は、以下の点に注意してください。

  • あなたのタグは公開されます。だれもがタグを見ることができます。
  • 個々のタグにスペースを入れることはできません。スペースは複数のタグを入力するときに使います。"big brother" ではなく "bigbrother" を使ってください。
  • 逆の意味を表すタグは、先頭に "!" を付けます。たとえば、"!funny" は "おもしろおかしくない" を意味します。
  • タグは簡潔にしてください。
  • タグの例をいくつか用意しています。気に入ったらお使いください。
  • 「タグ」をクリックしてタグを保存するのを忘れないでください。
  • 保存した後で、現在のタグのリストがテキストフィールドに戻されます。それを消去してもう一度「タグ」をクリックすると、古いタグは解除されます。タグをさらに追加するには、リストに追加して「タグ」をもう一度クリックします。
  • タグはすべて小文字でなければならず、句読点は使えません。数字は使えますが、先頭を数字にすることはできません。"Web 2.0" であれば、短縮して "web20" というタグにします。文字数の上限は 64 文字です。（訳注: スラッシュドット ジャパンの場合は、日本語のタグをつけることができます）
  • できるだけ複数形を避け、"harddrives" ではなく "harddrive" とします。

我々はこれに胸を躍らせ、このシステムに大きな可能性を見いだしています。記事に対するユーザのフィードバックからコメントのモデレーションまで、システムの可能性は無限であり、すべてはあなたの参加と、我々のデータベースハードウェアにかかっています。

タグは客観的ですか、それとも主観的ですか?どのようなタグでも好きに使えますか?

他のタグ付けシステムでは、ユーザが好きなタグを作れるようになっていて、客観的な意味の問題をあいまいにしています。そのため "foo" というタグはユーザごとに意味がまちまちで、システムにとっては何の意味も持たず、単なる識別名に過ぎません。

我々は、タグの上に次世代のモデレーションを構築しようとしています。つまり、あなたの名前空間に侵入するということです。一部のタグは、最初から (またはすぐ後から) システムに対する実質的な影響を持つようになります。記事のタグ付け機能は、"重複" や "typo" などのタグを認識します。今後コメントのタグ付けを導入するときは、この機能に "荒らし" や "参考になる" を学習させる予定です。これらのタグはシステム内で意味を持つことができます。我々が予想もしなかったタグの使い方をあなたが考え付いたときは、それをシステムに取り込みます。

ただし、あなたのタグはあなたに属するのではなく、全員に属します。我々は必要と考える変更を加える権利を留保します。このシステムはまったくの実験段階なので、これがどのような結果をもたらすかはっきりとはわかりません。ある日、すべての "イヌ科の動物" を統合して "イヌ" というタグに変更するかもしれません。何らかのシステムタグを定義し、別のタグを強奪するかもしれません。アクティブに開発されているシステムとはこういうものであることを念のために警告しておきましょう。

よそでは、タグはあなた個人のもので、ぞんざいに扱ってもだれも気にしません。スラッシュドットでは、タグを不正に使用すると負の結果をもたらすことがあります。これが正確に何を意味するかは、今のところわかりません。

たしかに、これはかなり恣意的です。今後の展開に合わせてポリシーを策定していきます。さしあたっての取り決めは、誠意を持ってタグを付けていただき、不正使用があれば、我々が誠意を持って対処するということです。

どのようなタグが定義されていますか?

重複 は、スラッシュドットのストーリーが実際に以前のスラッシュドットのストーリーと重複していて、新しい情報を何も提供していない場合にのみ使います。

(ところで、重複 または !重複 タグの使い方を誤ると、我々のシステムにおけるあなたのユーザアカウントの効力が今後何年にもわたって確実に低下します。正確にすばやく使えば、それと逆の効果があります。)

typo (入力ミス) は、スラッシュドットのストーリー記事にスペルまたは文法上のエラーがあるか、不正な形式のリンクなどの HTML の誤りがある場合に使います。質問にスペルミスの単語が含まれるストーリーにはタグを付けないでください。

これらのタグは、我々に問題を直ちに通知します (ただし、上部のタグリストには表示されません)。

スラッシュドットブックマークとは何ですか?

スラッシュドットブックマークは、URL を保存しておいて、スラッシュドットのコミュニティと共有したり、スラッシュドットの編集者にタレコんで掲載を検討してもらうための手軽な場所です。

何よりもまず、ブックマークは公開されるということを忘れないでください。あなたがブックマークしたものは、すべてスラッシュドットのユーザ全員に公開されます。自分のブックマークにタグを付けて、後で見つけやすくすることもできます。

ブックマークした URL は、スラッシュドットのタレコミページに直接タレコむこともできますが、もっといいのは、それに関する日記のエントリを書くことです。スラッシュドットの日記のエントリは、メインページ掲載候補としてスラッシュドットの編集者に自動的にタレコむことができるので、ブックマークはストーリーをすばやくタレコむのに打ってつけの方法です。

スラッシュドットでブックマークするよう自分の読者に促すにはどうすればいいですか?

あなたのサイトの Web ページにまだ "Digg It" や "Add to Del.icio.us" というリンクが散乱していなければ、ページをスラッシュドットに自動的にタレコむためのバッジまたはリンクを追加することができます。十分な数の読者がクリックすると、あなたのページが編集者の目にとまります。早くて安上がりな submit.pl のようなものです。

バッジはこのように表示されます。ページをタレコむだけでなく、読者はタレコまれたページの現在の人気度を見たり、ページに投票したりすることができます。次の HTML のブロックを任意のページに挿入してください。

<script src="http://slashdot.jp/slashdot-it.js" type="text/javascript"></script>

詳細と例については、「バッジの手引き」を参照してください。

ブックマークを共有するには?

自分のブックマークは /my/bookmarks から利用できます。他のユーザのブックマークには、そのユーザのホームページからアクセスすることができます。私のブックマークを見る場合は、/~CmdrTaco/bookmarks にアクセスします。

スラッシュドットのユーザがどのようなタグを付けているかを見るには?

http://slashdot.jp/tags で、最も広く使われているタグの一部を見ることができます。それぞれのタグは、ストーリーおよびブックマークに関する読者の何らかの考えを示しています。分類系のタグの多くはミニ検索のような役割を果たし (web20、google、globalwarming、diebold、bush、rumor、xbox)、コメント系のタグではスラッシュドットのユーザの意見を垣間見ることができます (itsatrap、fud、snakeoil、cool、finally、whocares、それに「Jeopardy」(訳注: アメリカのクイズ番組) 形式の回答 yes と no)。

自分自身のタグは /my/tags で見ることができ、私のタグは /~CmdrTaco/tags に表示されます。

我々は常に改良を続けており、RSS フィード、タグクラウド、見やすい表示形式も今後登場する予定です。
